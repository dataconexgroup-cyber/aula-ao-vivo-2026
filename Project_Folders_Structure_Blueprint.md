# Project Folder Structure Blueprint

## 1. Structural Overview

The `aula-ao-vivo-2026` project is a modern web application built using **React**, **TanStack Start** (which incorporates TanStack Router and React Query), **Vite**, and **Tailwind CSS**. It relies heavily on server-side functions (via TanStack Start) and integrates with **Supabase** for database/authentication.

The organization principle is largely **feature and layer-based**, where:
- Frontend routes are encapsulated in `src/routes/`.
- Reusable UI components are stored in `src/components/ui/` (based on shadcn-ui).
- Server-side logic and database functions are kept in `src/server/`.
- External integrations (like Supabase client initialization and auth) are stored in `src/integrations/`.

This setup enforces a clear separation of concerns, keeping the presentation layer separate from server logic and infrastructure configurations.

## 3. Key Directory Analysis

#### UI Project Structure (React / TanStack Start)

- **Component Organization (`src/components/`)**:
  - `src/components/ui/`: Contains atomic, reusable UI components generated by shadcn-ui (e.g., `button.tsx`, `dialog.tsx`).
  - Feature-specific components could be placed directly in `src/components/` or within subfolders named after features.

- **Routing Organization (`src/routes/`)**:
  - Based on TanStack Router's file-based routing.
  - `__root.tsx`: The primary layout wrapper that wraps all other routes.
  - Page routes (e.g., `index.tsx`) contain the UI for specific URLs.

- **API/Server Integration (`src/server/`)**:
  - Server-side functions executed securely via TanStack Start.
  - Contains database operations and sensitive logic (e.g., `survey.functions.ts`).

- **External Integrations (`src/integrations/`)**:
  - Used for encapsulating third-party service connections.
  - `supabase/`: Holds standard client initializers, server clients, auth middleware, and typed database definitions (`types.ts`).

- **Utility Functions (`src/lib/`)**:
  - `utils.ts`: Contains general-purpose helpers (like `cn` for Tailwind class merging).

## 4. File Placement Patterns

- **Configuration Files**:
  - Root directory holds all primary configs: `vite.config.ts`, `tsconfig.json`, `package.json`, `bunfig.toml`, `wrangler.jsonc`, `eslint.config.js`.
  - `.env` stores environment-specific secrets.
  
- **Model/Entity Definitions**:
  - Database types generated by Supabase are stored in `src/integrations/supabase/types.ts`.
  
- **Business Logic**:
  - Server-side operations and data mutations are placed in `src/server/`.
  
- **Interface Definitions**:
  - Component props are typically defined within the same file as the component. Shared TypeScript interfaces can be placed in a `types` folder or alongside their respective modules.

## 5. Naming and Organization Conventions

- **File Naming Patterns**:
  - `kebab-case.tsx` is heavily used for components and hooks (e.g., `alert-dialog.tsx`, `use-mobile.tsx`).
  - `.ts` is used for logic files, and `.tsx` for React components.
  - Route files use specific TanStack Router conventions (e.g., `__root.tsx` for layout routes, `index.tsx` for index routes).
  
- **Folder Naming Patterns**:
  - All lowercase, pluralized where applicable (`components`, `routes`, `hooks`).

- **Organizational Patterns**:
  - Shadcn-ui pattern: keep all primitive UI components isolated in `src/components/ui/` to prevent mixing business logic with core UI building blocks.

## 6. Navigation and Development Workflow

- **Entry Points**:
  - `src/router.tsx`: Sets up the TanStack Router instance.
  - `src/routes/__root.tsx`: The main application shell/layout.
  - `src/routes/index.tsx`: The starting point for the visual application.

- **Common Development Tasks**:
  - **Adding a new page**: Create a new `.tsx` file in `src/routes/` following TanStack Router conventions. The `routeTree.gen.ts` will update automatically.
  - **Adding a new UI component**: Use `npx shadcn-ui@latest add [component]` or manually place it in `src/components/ui/`.
  - **Adding server-side logic**: Create a new file in `src/server/` and export TanStack server functions.

## 7. Build and Output Organization

- **Build Configuration**:
  - `vite.config.ts` handles the Vite build process.
  - `package.json` defines scripts (`npm run dev`, `npm run build`).

- **Environment-Specific Builds**:
  - Handled by `.env` variables and Vite modes. 
  - Cloudflare deployment configured via `wrangler.jsonc` and `@cloudflare/vite-plugin`.

## 8. Technology-Specific Organization

#### React & Node.js Specific Structure Patterns
- **Package Management**: Managed via `npm` (with `package-lock.json`) and `bun` (with `bun.lockb`).
- **Module Organization**: Uses ESM (ECMAScript Modules) as defined by `"type": "module"` in `package.json`.
- **Third-party Dependency Management**: Dependencies are strictly listed in `package.json` with a clear split between dev and prod dependencies.

## 9. Extension and Evolution

- **Extension Points**:
  - New features should be organized in their respective routes within `src/routes/`.
  - Complex features can have their own subdirectories within `src/components/` (e.g., `src/components/dashboard/`).
  
- **Scalability Patterns**:
  - As the application grows, consider migrating from flat `src/components/` to a feature-based structure (e.g., `src/features/survey/components/`).

## 10. Structure Templates

- **New Route Component Template** (`src/routes/my-route.tsx`):
  ```tsx
  import { createFileRoute } from '@tanstack/react-router'

  export const Route = createFileRoute('/my-route')({
    component: MyRouteComponent,
  })

  function MyRouteComponent() {
    return <div>My New Route</div>
  }
  ```

- **New Server Function Template** (`src/server/my-feature.functions.ts`):
  ```typescript
  import { createServerFn } from '@tanstack/react-start'

  export const myServerAction = createServerFn({ method: 'POST' })
    .handler(async ({ data }) => {
      // server-side logic here
      return { success: true };
    })
  ```

## 11. Structure Enforcement

- **Structure Validation**: 
  - ESLint (`eslint.config.js`) enforces code quality and can be configured to restrict import patterns.
  - TypeScript (`tsconfig.json`) ensures strict type checking across paths (using `paths` for module resolution if configured).
- **Auto-generation**:
  - `src/routeTree.gen.ts` is strictly auto-generated by `@tanstack/router-plugin` and should not be manually edited.
